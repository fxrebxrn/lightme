# LightMe Telegram Bot

Telegram-бот для мониторинга графиков отключений электроэнергии (ДТЕК/ЦЕК), подписок на очереди, сравнения двух очередей и автоматических уведомлений.

## Что умеет бот

- Показывает графики по компаниям **ДТЕК** и **ЦЕК**.
- Поддерживает сохранение до 5 очередей на пользователя.
- Отправляет 4 типа уведомлений:
  - отключение,
  - включение,
  - напоминание за 10 минут до отключения,
  - напоминание за 10 минут до включения.
- Позволяет сравнивать 2 очереди и находить интервалы «совместного света».
- Поддерживает 2 языка интерфейса: **украинский** и **русский**.
- Имеет админ-команды для загрузки расписаний, техработ, рассылок и обслуживания БД.

## Быстрый старт (локально)

### 1) Клонирование

```bash
git clone <YOUR_REPO_URL>
cd lightme
```

### 2) Виртуальное окружение

Linux / macOS:

```bash
python3 -m venv .venv
source .venv/bin/activate
```

Windows (PowerShell):

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
```

### 3) Установка зависимостей

```bash
pip install -r requirements.txt
```

### 4) Переменные окружения

Обязательная переменная:

- `API_TOKEN` — токен Telegram-бота от @BotFather.
- `ADMIN_ID` — ID пользователя Telegram который администрирует бота.
- `CHANNEL_USERNAME` — username Telegram-канала бота.
- `CHANNEL_URL` — URL Telegram-канала бота.
- `SUPPORT_USERNAME` — username пользователя Telegram который отвечает за тех. поддержку бота.
- `DONATE_URL` — URL на поддержку бота (донат).
- `DATABASE_NAME` — имя базы данных с расширением (по умолчанию .db).
- `DATABASE_PATH` — путь к базе данных.

Пример (Linux/macOS):

```bash
export BOT_TOKEN="123456:ABC..."
```

Пример (Windows PowerShell):

```powershell
$env:BOT_TOKEN="123456:ABC..."
```

### 5) Запуск

```bash
python main.py
```

При первом запуске автоматически создаются таблицы в SQLite и поднимается планировщик уведомлений.

## База данных

Бот использует SQLite.

- Если существует папка `/app/data`, база хранится в `/app/data/database.db` (удобно для Docker/Volume).
- Иначе используется файл `database.db` в корне проекта.

Основные таблицы:

- `users` — подписки пользователей на очереди;
- `user_prefs` — язык и настройки уведомлений;
- `schedules` — графики отключений;
- `settings` — системные настройки;
- `bot_settings` — флаги режима техработ/аварийных отключений.

## Пользовательские команды

- `/start` — старт, выбор языка, главное меню.
- `/menu` — открыть главное меню.
- `/sched` — открыть просмотр графиков.
- `/add` — добавить очередь.
- `/subs` — мои очереди.
- `/support` — контакты и ссылка на донат.

> Большая часть навигации реализована кнопками и inline-кнопками в чате.

---

## Админ-команды

- `/ahelp` — справка по админ-командам.
- `/amenu` — админ-меню с быстрыми кнопками.
- `/upload ...` — загрузка графика (текстом).
- `/news ...` — рассылка пользователям.
- `/stats` — статистика бота.
- `/getdb` — отправка БД администратору.
- `/techon` / `/techoff` — вкл/выкл режим техработ.
- `/avaron` / `/avaroff` — флаг аварийных отключений.

Права администратора задаются через `ADMIN_ID` в `config.py`.

## Формат загрузки графика (`/upload`)

### Вариант 1: обычный график

```text
ДТЕК 29.01.2026
Черга 1.1
00:00-04:00
12:00-15:00
Черга 1.2
-
```

Поддерживается несколько интервалов для одной очереди.

### Вариант 2: массовое `empty` для всех очередей

```text
/upload ДТЕК/ЦЕК 29.01.2026
-
```

Это помечает все очереди на дату как «без отключений».

## Развертывание (Docker / Railway)

Для production-сценария рекомендуется:

1. Передать `BOT_TOKEN` через переменные окружения.
2. Подключить volume в `/app/data` для сохранности `database.db`.
3. Запускать процесс командой `python main.py`.
